# Integrity-of-Data-in-Amazon-S3
Checking the Integrity of Data in Amazon S3 with Additional Checksums

## Tools & Services Used
- Amazon S3
- AWS Management Console
- SHA-256 Checksum
- Terminal (shasum)

## Steps Performed

### 1. Create an S3 Bucket
- Created a globally unique S3 bucket
- Used default settings (Block Public Access enabled)

### 2. Upload File with Checksum
- Uploaded a file to S3
- Selected storage class as standard
- left server-side encryption as default
- Selected the Checksum function as #SHA256

### 3. Verify Checksum
- Opened file properties in S3
- Located the checksum value generated by AWS
- Compared with local checksum using:

```bash
shasum -a 256 IMG_5653.jpg | cut -f1 -d\ | xxd -r -p | base64
